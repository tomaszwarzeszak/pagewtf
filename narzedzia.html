3.02 Kanban

Kanban - „spis widoczny” – sposób planowania swojej pracy przydatne zarówno w projekcie
jak i w wielu innych czynnościach życia codziennego.
Plan pracy zazwyczaj zapisywany jest na tablicy lub w specjalnie przygotowanych do tego
aplikacjach takich jak np. „Trello”.

Tablicę dzielimy na min. 3 kolumny:
- Pierwsza „Todo” - zapisujemy poszczególne kroki lub zadania do wykonania
np. części projektu
- Druga „Doing” - zamieszczamy rzeczy, które wykonywane są w danym momencie
(ważne aby w tej kolumnie była jak najmniejsza ilość zadań do wykonania w danym momencie najlepiej jedna)
- Trzecia „Done” –w tej kolumnie zamieszczamy rzeczy, które zostały już zrealizowane.

3.05 Jak działa Internet i przeglądarka

http – jest to protokół, którego używają przeglądarki do wczytywania stron z serwera.
https – jest to protokół za pomocą, którego przeglądarki wczytują pliki z serwera ten rodzaj protokołu jest szyfrowany i obecnie najbardziej zalecany do używania.

URL – Uniform Resource Locator – Jest to adres, który wpisujemy w przeglądarce aby wczytać pliki z serwera.

Proces wczytywania strony:
- wpisuję w pasku adresu np. cotenfrontend.pl
- przeglądarka szuka serwera cotenfrontend.pl [DNS]
- po znalezieniu serwera odpytuje go o index
- serwer wysyła do przeglądarki zawartość html
- przeglądarka wczytuje html i szuka wszystkich dodatkowych zasobów: style, skrypty, obrazki itp. i dla każdego z zasobów wykonuje zapytanie.
- Przeglądarka parsuje i renderuje stronę.

W narzędziach developerskich w przeglądarce w zakładce Network pokazany jest ruch sieciowy podczas wczytywania strony.

Kody odpowiedzi http:
- 200 – jest ok wszystko działa.
-30x – Szukasz w złym miejscu przekieruję Cię w dobre miejsce bo wiem gdzie.
-404 – Błąd – nie znaleziono tego czego szukasz.
- 50x – Błąd wewnętrzny serwera.

Root „/” – ten slash mówi w, którym miejscu przeglądarka powinna szukać zasobów.

3.07 Terminal

Terminal w systemie Windows odpalamy jako „Wiersz Poleceń”

Komendy w terminalu w systemie Windows:
- cd C:\ - przejście do głównego katalogu na dysku C
- dir – wypisanie zawartości katalogu
- cd .. – przejście do katalogu wyżej
- cd „nazwa-katalogu” – przejście do katalogu o nazwie „nazwa-katalogu”
- code . – uruchomienie Visual studio code w aktualnym katalogu
- cd – wyświetlenie aktualnej ścieżki
- mkdir „nazwa katalogu” – stworzenie katalogu o nazwie nazwa-katalogu

4.02 Jest Git

GIT- Jest to system kontroli wersji projektowych. Służy do przechowywania oraz dzielenia
się projektami grupowo, system ten pozwala na wprowadzaniu aktualizacji projektu oraz
powrotu do stanu przed aktualizacją.

System Git składa się z repozytoriów, które inaczej mówiąc są naszymi projektami gdzie
zawarte są wszystkie pliki projektu.

System GIT należy pobrać ze strony internetowej http://git-scm.com oraz przeprowadzić
konfigurację w wierszu polecenia.

Commit –Stan projektu oraz opis wszystkich zmian jakie dokonaliśmy od ostatniej aktualizacji.

Komendy:
- git config --global user.name ``Imię Nazwisko” ; git config – global user.email ``podać adres e-mail”- Komedy służące do przeprowadzenia konfiguracji systemu GIT.
- git config –get user.name ; git config –get user.email – Komendy do sprawdzenia stanu konfiguracji.
- git init – Tworzenie nowego repozytorium
- git status – wypisuje rzeczy znajdujące się w repozytorium
- git add . – służy do dodania wszystkich plików znajdujących się na dysku a nie będących jeszcze w repozytorium
- git commit –m „ opis zmiany” – służy do zacommitowania zmian w naszym repozytorium.
- gitk – polecenie, które uruchamia okno z całą historią poleceń naszego repozytorium.

5.06 U mnie (nie) działa

„W życiu każdego programisty i programistki przychodzi taki motyw kiedy nie działa” M.Korsan

Proces szukania błędów i naprawiania ich w naszym kodzie to Debugging.

Co należy zrobić gdy nie działa:
- Sprawdzić czy jest zapisany plik.
- Sprawdzić czy edytujemy poprawny plik
- Sprawdź czy odświeżyłeś przeglądarkę (Ctrl + f5)
- Sprawdź czy nie ma błędów w edytorze/devtoolsah
- CZYTAJ BŁĘDY
- Zajrzyj w zakładkę Network
- „Pogadaj z Kaczką”
- Sprawdź w Google. (bierz pod uwagę porady z datą max. do roku wstecz jedna z najbardziej rekomendowanych stron do szukania błędów to stackoverflow).
