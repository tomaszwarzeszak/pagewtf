3.02 Kanban

Kanban - „spis widoczny” – sposób planowania swojej pracy przydatne zarówno w projekcie
jak i w wielu innych czynnościach życia codziennego.
Plan pracy zazwyczaj zapisywany jest na tablicy lub w specjalnie przygotowanych do tego
aplikacjach takich jak np. „Trello”.

Tablicę dzielimy na min. 3 kolumny:
- Pierwsza „Todo” - zapisujemy poszczególne kroki lub zadania do wykonania
np. części projektu
- Druga „Doing” - zamieszczamy rzeczy, które wykonywane są w danym momencie
(ważne aby w tej kolumnie była jak najmniejsza ilość zadań do wykonania w danym momencie najlepiej jedna)
- Trzecia „Done” –w tej kolumnie zamieszczamy rzeczy, które zostały już zrealizowane.

3.02 WTF: HTML II

Nowe znaczniki:
- <br/> - łamanie linii, przenosi kursor do nowej linii.
- <b> - znacznik do pogrubienia tekstu, który nie ma znaczenia semantycznego służy do wyróżnienia tekstu z otaczającego.
- <i> - znacznik do stworzenia pochylonego tekstu, który nie ma znaczenia semantycznego służy do wyróżnienia tekstu z otaczającego.
- <strong> - znacznik do pogrubienia tekstu , który ma znaczenie semantyczne, podkreśla wysoką istotność i poważność frazy w nim zawartej.
- <Em> znacznik do stworzenia kursywy w tekście, mający znaczenie semantyczne podkreśla, że tekst może mieć inne znaczenie (np.ironia).
- <a href=”” target=””>opis łącza</a> - linki/ łącza wewnątrz dokumentu.
- <element id=”mój-identyfikator”> - atrybut id – identyfikuje element (przydaje się do nawigacji w obrębie dokumentu).
Aby odwołać się do id w obrębie dokumentu należy zapisać <a href=”#moj-identyfikator”>opis łącza</a>
- <nav> - znacznik w, którym zawieramy nawigację strony.
- <img src=”” alt=””> - dodawanie obrazku na stronę.

3.03 Opowieść o wężu, kebabie, wielbłądzie i Pascalu

„Najtrudniejszą rzeczą w programowaniu nie jest wymyślanie algorytmów, testowanie aplikacji czy pisanie kodu, lecz wymyślanie nazw dla rzeczy” M.Korsan

Reguły dotyczące nazywania rzeczy:
- nazwy zapisujemy bez polskich znaków,
- nie używamy spacji,
-najlepiej używać angielskich słów
-trzymamy się jednego schematu

Konwencje nazewnictwa:
- camelCase – nazwa pochodzi od budowy wielbłąda zapisujemy pierwsze słowo małą literą a każde kolejne rozpoczynamy z dużej.
- kebab-case – nazwa wzięta z szaszłyka zapisujemy każde słowo z małej litery rodzielając je znakiem „-”.
- snake_case – nazwa wzięta z węża snake zapisujemy go z małej litery oddzielając każde słowo znakiem „_”.
-PascalCase – nazwa wzięta z języka Pascal zapisujemy każde słowo z dużej litery.

3.04 WTF: CSS II

Jednym z sposobów załączania stylów css jest załączenie do znacznika jako atrybut zwany „inline”. Polega on na wpisywaniu własności wewnątrz znacznika.
Nie jest to zalecany sposób dylowania strony.
Kolejnym ze sposobów załączania stylów jest stylowanie poprzez tzw. „Prawdziwe selektory”.
Jednym z najbardziej zalecanych i popularnych sposobów jest stylowanie poprzez dodawanie atrybutu „class” lub „Id” do wybranego znacznika w arkuszu HTML.

W arkuszu stylów klasę zapisujemy :
.nazwa-klasy{
właściwiość: wartość;
}

Można używać tej samej klasy w kilku znacznikach.

W arkuszu stylów id zapisujemy:
#name{
właściwość: wartość;
{

Id używamy tylko jeden raz w arkuszu HTML.

Selektoty możemy z sobą łączyć zapisując je po spacji, nazywamy to „specyficznością CSS”.

Własności CSS na stronie wczytywane są wg. specyficzności:
-!important – jest to własność zapisywana bezpośrednio przy własności w arkuszu CSS.
- style=”” – własność zapisywana wewnątrz znacznika HTML.
- #name – id zapisywane w znaczniku HTML a stylowane w arkuszu CSS.
-.name - klasa zapisywana jest w znaczniku HTLML a stylowana w arkuszu CSS.
- element – zapisywany jest w sekcji body w arkuszu HTML.

Jeżeli specyficzność jest równa wygrywa selektor bliżej elementu.

Najbardziej zalecaną formą jest dodawanie klasy do wszystkich elementów i unikanie zagnieżdżeń.

3.05 Jak działa Internet i przeglądarka

http – jest to protokół, którego używają przeglądarki do wczytywania stron z serwera.
https – jest to protokół za pomocą, którego przeglądarki wczytują pliki z serwera ten rodzaj protokołu jest szyfrowany i obecnie najbardziej zalecany do używania.

URL – Uniform Resource Locator – Jest to adres, który wpisujemy w przeglądarce aby wczytać pliki z serwera.

Proces wczytywania strony:
- wpisuję w pasku adresu np. cotenfrontend.pl
- przeglądarka szuka serwera cotenfrontend.pl [DNS]
- po znalezieniu serwera odpytuje go o index
- serwer wysyła do przeglądarki zawartość html
- przeglądarka wczytuje html i szuka wszystkich dodatkowych zasobów: style, skrypty, obrazki itp. i dla każdego z zasobów wykonuje zapytanie.
- Przeglądarka parsuje i renderuje stronę.

W narzędziach developerskich w przeglądarce w zakładce Network pokazany jest ruch sieciowy podczas wczytywania strony.

Kody odpowiedzi http:
- 200 – jest ok wszystko działa.
-30x – Szukasz w złym miejscu przekieruję Cię w dobre miejsce bo wiem gdzie.
-404 – Błąd – nie znaleziono tego czego szukasz.
- 50x – Błąd wewnętrzny serwera.

Root „/” – ten slash mówi w, którym miejscu przeglądarka powinna szukać zasobów.

3.06 WTF: JavaScript

JS = JavaScript – Jest to język skryptowy programowania webowego. Na stronie internetowej odpowiada za interakcje i może wpływać na arkusz CSS, HTML oraz content strony.

Język JS składa się z:
- Wartości – dla języka JS jest jak liczby w matematyce
- Operacji porównywania -
- Literały – specjalny ciąg znaków dzięki, któremu możemy wypisać w konsoli tekst razem ze zmiennymi.
- Zmienne – połączenie między nazwą, którą wprowadzamy a wartością.
-Obiekty
-Funkcje

Kod JS podpinamy do strony za pomocą znacznika <script> w arkuszu HTML lub tworzymy oddzielny plik skryptowy z rozszerzeniem .js

Polecenia języka JS:
-console.log(‘Siema!’); - to polecenie wyświetla nam wartości w narzędziach developerskich w karcie Console.

3.07 Terminal

Terminal w systemie Windows odpalamy jako „Wiersz Poleceń”

Komendy w terminalu w systemie Windows:
- cd C:\ - przejście do głównego katalogu na dysku C
- dir – wypisanie zawartości katalogu
- cd .. – przejście do katalogu wyżej
- cd „nazwa-katalogu” – przejście do katalogu o nazwie „nazwa-katalogu”
- code . – uruchomienie Visual studio code w aktualnym katalogu
- cd – wyświetlenie aktualnej ścieżki
- mkdir „nazwa katalogu” – stworzenie katalogu o nazwie nazwa-katalogu
