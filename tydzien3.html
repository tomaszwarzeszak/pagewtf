4.02 Jest Git

GIT- Jest to system kontroli wersji projektowych. Służy do przechowywania oraz dzielenia
się projektami grupowo, system ten pozwala na wprowadzaniu aktualizacji projektu oraz
powrotu do stanu przed aktualizacją.

System Git składa się z repozytoriów, które inaczej mówiąc są naszymi projektami gdzie
zawarte są wszystkie pliki projektu.

System GIT należy pobrać ze strony internetowej http://git-scm.com oraz przeprowadzić
konfigurację w wierszu polecenia.

Commit –Stan projektu oraz opis wszystkich zmian jakie dokonaliśmy od ostatniej aktualizacji.

Komendy:
- git config --global user.name ``Imię Nazwisko” ; git config – global user.email ``podać adres e-mail”- Komedy służące do przeprowadzenia konfiguracji systemu GIT.
- git config –get user.name ; git config –get user.email – Komendy do sprawdzenia stanu konfiguracji.
- git init – Tworzenie nowego repozytorium
- git status – wypisuje rzeczy znajdujące się w repozytorium
- git add . – służy do dodania wszystkich plików znajdujących się na dysku a nie będących jeszcze w repozytorium
- git commit –m „ opis zmiany” – służy do zacommitowania zmian w naszym repozytorium.
- gitk – polecenie, które uruchamia okno z całą historią poleceń naszego repozytorium.

4.04 Życie w pudełku: Display, wymiary i overflow

Nowe znaczniki:
-<div> - jest to znacznik. który nie ma znaczenia semantycznego, używany jest w sytuacji gdy chcemy coś „opakować” (tylko wtedy gdy nie istnieje odpowiedni semantyczny znacznik).

Własność CSS:
- display: Block – własność, która powoduje przeniesienie danego elementu (pudełka) do nowej lini.
- display: inline – własność , za pomocą której elementy otrzymują rozmiar zawartości bloku oraz ustawiają się jednen obok drugiego.
- display: inline-block – własność ta nadaje elementowi cechy inline, jednak możemy nadawać dodatkowe cechy, które są dostępne dla własności blokowych a nie są dostępne dla inline.
- width/height – własność za pomocą, której możemy ustalać wysokość oraz szerokość zarówno całej strony jak i wskazanego elementu. Własności te nie są dostępne dla „display:inline”
- max-width/max-height – wartości te powodują, że elementy na stronie dopasowują się szerokością i wysokością do okna przeglądarki w której są wyświetlane.
- min-width/ min-height – określa minimalną wysokość oraz szerokość jaką musi mieć dany element na stronie.
- overflow: hidden – powoduje odcięcie treści która wychodzi za wcześniej zdeklarowane wartości heisgh i width.
- overflow: visible – własność za pomocą, której element wychodzi poza zdeklarowaną wysokość oraz szerokość.
- overflow: auto- do treści która wychodzi poza obszar boxu dokładany jest scroll.

4.05 Jednostki

W zapisie własności używamy jednostek:
-0 – jest to jednostka w której nie musimy zapisywać %, px itp.
-px- piksele są jedną z najpopularniejszych używanych jednostek na stronie internetowej, ponieważ ekran jest zbudowany z pikseli.
-% - procent- jest to jednostka która nadaje wielkość elementu wg % elementu w którym się znajduje.
- vw – viewport width – jest to jednostka która ustawia % całego okna a nie elementu(rodzica)
- vh – viewport height – jest to jednostka która ustawia % wysokości wg. całego okna.

4.06 Życie w pudełku: Box model (pudding/border/margin)

Box model są to elementy(prostokąty) opakowujące poszczególne części strony.

Box model składa się z:
-padding –jest to margines wewnętrzny elementu odpowiada za to o ile mają być oddalone elementy które znajdują się w środku elementu, który ma padding.
Jego wartości zapisujemy zgodnie z ruchami wskazówek zegara (top/right/bottom/left).
Są również wartości „lustrzane” czyli jeżeli zapiszemy dwie wartości to pierwsza będzie się odnosić do góra/dół a druga prawo/lewo.
- border – jest elementem który tworzy „obramowanie” elementu.
Przy definicji border podajemy width/style/color.
- border radius – pozwala na zaokrąglenie elementu.
-margin – jest to margines zewnętrzny, który występuje poza elementem odpowiada za to o ile możemy się odsunąć od innych elementów na stronie.
Sposób zapisywania jest identyczny jak w przypadku pudding.
-margin: …px auto; - powoduje wyśrodkowanie elementu.
Margines automatyczny działa tylko wtedy gdy szerokość elementu jest zdefiniowana.

4.07 Życie w pudełku: box-sizing

Domyślną wartością Box Sizing to wartość „content-box” mówi nam ona to, że jeżeli zdefiniujemy sobie szerokość contentu to nie wpływa to na pudding, border, margin

-„border box”odpowiada za automatyczne dopasowanie contentu w środku elementu w momencie dodania wartości padding, border całość elementu wprowadzona w wartości „width” zostanie zachowana.

-„content box” powoduje automatyczne powiększenie elementu o wartość pudding oraz border nie naruszając wprowadzonych wcześniej wartości width oraz height contentu na naszej stronie.

4.08 Życie w pudełku: position

Position- Jest to wartość która pozwala nam określić dokładną pozycję elementu na stronie co do Piksela.

Własności position:
- position: static – jest to domyślna własność elementu
- position: relative – sprawia, że względem tego elementu gdzie jest on zdefiniowany będą przesuwanie elementy w własności „absolute”
- position: absolute – pozwala za pomocą własności „top, right, bottom, left” przesuwać nasz element na stronie co do piksela.
- z-index – za pomocą tego atrybutu definiujemy kolejność elementów wyświetlonych w przypadku gdy elementy na siebie nachodzą.
- position: fixed – posiada bardzo zbliżone własności do „absolute” różni się tym, w momencie przewijania strony element zostaje na swoim miejscu nie znika z ekranu, przy tej własności należy zdefiniować szerokość elementu.
- position: sticky – polega na przyklejeniu w momencie gdy krawędź przy skrolowaniu uderza w krawędź okna. (rodzic tego elementu nie może mieć własności overflow)

4.09 Życie w pudełku: float

Float- służy do opływania elementów na stronie.
Clear - służy do „czyszczenia” treści i rozszerzenia boxu do wielkości elementu zawierającego float.

Float nie służy do budowania layoutów.

4.10 JS: Typy danych i immutability

Zmienna- Jest to połączenie z naszym wszechświatem wartości.

Typy zmiennych w JS:
-LET- zmienna, których połączenie możemy przepinać, ale definiujemy ją tylko raz
-CONST – jest to zmienna mająca stałe połączenie z jedną wartością
-VAR

Typy proste w JS (primitives)
- String – tekst, otaczamy ‘ lub „
- Number – liczba
- Bigint – duża liczba większa niż 2 do 53
-boolean –true lub false
-null – wartość celowo niezdefiniowana
-undefined – brak definicji
-*symbol – tworzy „unikalne opakowanie”

Typeof – wartość, która służy do sprawdzenia jakiego typu jest dana wartość.

Immutability – Oznacza to, że nie możemy zmieniać wartości primitive values w JS. To co zmieniamy to połączenie między nimi a zmiennymi.

4.12 Block – Element – Modifier

BEM – Sposób zapisywania klas css ułatwiający pracę nad całym projektem.

B- Block - nadrzędny element
E- Element - część bloku (nie musi być bezpośrednim dzieckiem)
M- Modifie - modyfikator jeśli element występuje w kilku wariacjach tworzymy dodatkową klasę.

BEM zapisywany jest na wzór dłoni.
przykładowa klasa bazowa to class=”hand”, kolejnym poziomem jej dziećmi jest class=”hand__finger”, zaś dzieckiem tej klasy class=”hand__finger—thumb”

4.13 DOMowe przedszkole

DOM – Document Object Model – obiektowy model dokumenu. Czyli efekt wizualny instrukcji zapisanej w edytorze kodu w języku HTML.

Aby skorzystać z wartości DOM w JS musimy podpiąć się za pomocą metody:
„dokument.querySelector(`.selector’) ”

Aby dostać się do elementu w JS najlepiej jest to zrobić za pomocą stworzenia nowego selektora wg. BEM dodając jako modyfikator „–js”

Po stworzeniu zmiennej z podpięciem się do elementu możemy dobrać się oraz zmienić wartość wprowadzoną w kodzie HTML za pomocą metody innerHTML:
„(nazwa zmiennej).innerHTML = ‘wartość którą ma zostać zastąpiony HTML’.
